Car Sales Database Design

SQL Server relational database design project

SQL Server Database Design ERD Data Modeling

Project Overview

This project focuses on transforming a flat car sales dataset into a structured relational database. The original dataset included vehicle sale transactions, customer information, dealer details, car specifications, regions and sale prices.

The goal was to design a normalized SQL Server database, define primary and foreign keys, separate repeated entities into dedicated tables, and create a database structure suitable for future analysis in SQL, Python and Power BI.

Dataset

The original dataset included 16 columns and 23,906 rows. For the public GitHub version, a 200-row sample dataset is included. Customer names and phone numbers were anonymized before publishing.

Database Structure

The database was designed around the main business entities involved in a car sale process. The final structure includes separate tables for customers, regions, dealers, companies, models, body styles, cars and sales.

  • Customers — stores customer name, gender and annual income.
  • Regions — stores dealer regions.
  • Dealers — stores dealer names and dealer numbers.
  • Companies — stores car manufacturers and brands.
  • Models — stores car model names.
  • Body Styles — stores body style categories.
  • Cars — stores car specifications and references to company, model and body style.
  • Sales — stores sale transactions connecting customers, dealers, regions and cars.

Tools & Technologies

  • Microsoft SQL Server

  • Relational database design

Key Features

  • Created a SQL Server database from a flat car sales dataset.
  • Separated repeated entities into normalized relational tables.
  • Defined primary keys and foreign key relationships.
  • Loaded lookup tables using distinct values from the staging dataset.
  • Connected sale transactions to customers, dealers, regions and cars.
  • Created an ERD to visualize the database structure.
  • Prepared a public anonymized dataset sample for GitHub.

Business Value

This database structure makes the sales data easier to manage, query and analyze. Instead of working with one large flat table, the data is organized into meaningful entities that can support future reporting, dashboards and business analysis.

What I Learned

Through this project, I practiced database planning, table design, data modeling, SQL Server syntax, primary and foreign key relationships, normalizing a flat dataset, and preparing a project for future analytical work.
